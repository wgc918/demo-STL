# unordered_multiset 实现分析

## 基本概念

- **核心特性**：unordered_multiset 是一种关联容器，存储键（keys），允许键重复，但不保证任何顺序。
- **底层实现**：基于哈希表实现，使用链地址法解决哈希冲突。
- **平均时间复杂度**：O(1) 的插入、删除和查找操作。
- **与 unordered_set 的区别**：允许键重复，insert 总是成功，返回 iterator 而非 pair<iterator, bool>。

## 数据结构

### 哈希表结构

- 使用动态数组作为桶数组，每个桶存储一个单向链表。
- 桶数量始终为 2 的幂，通过位掩码（m_mask）快速计算桶索引。
- 默认桶数量为 16，默认最大负载因子为 1.0。

### 节点结构

```cpp
struct Node {
    value_type value;      // 键
    Node*      next;       // 指向同桶下一个节点的指针
    uint64_t   hash_code;  // 键的哈希值缓存
};
```

- 每个节点存储键和其哈希值（缓存），避免重复计算哈希值。
- 节点通过 next 指针链接，形成单向链表。

### 桶数组

- `Node** m_table`：桶数组指针，每个元素指向桶内链表头。
- `size_type m_mask`：桶索引掩码，用于快速计算桶索引。
- `size_type m_bucket_count`：桶数量，始终为 2 的幂。

## 哈希策略

### 哈希函数

- 使用 `std::hash<Key>` 作为默认哈希函数。
- 用户可以提供自定义哈希函数作为模板参数。

### 桶索引计算

- 桶索引通过哈希值与掩码的按位与运算得到：`hash & m_mask`。
- 由于桶数量为 2 的幂，该操作等价于 `hash % bucket_count`。

### 负载因子

- 负载因子 = 元素数量 / 桶数量。
- 当 `load_factor() > max_load_factor()` 时，自动触发重哈希。
- 重哈希时桶数量翻倍（2 的幂）。

## 冲突处理

### 链地址法

- 使用链地址法（Separate Chaining）处理哈希冲突。
- 同一桶内的元素通过单向链表链接。
- 插入时，将新节点插入到链表的适当位置。

### 重复键处理

- 与 unordered_set 不同，允许存储多个相等的键。
- 插入重复键时，新节点插入到桶中最后一个相等键的后面，保持相等键在链表中连续。
- 这确保了 `equal_range()` 返回的范围内所有元素都是匹配的。

## 迭代器设计

### 全局迭代器

- `iterator` 和 `const_iterator` 支持前向遍历整个容器。
- 迭代器通过 `m_node->next` 移动到下一个节点。
- 当到达桶末尾时，迭代器会查找下一个非空桶。

### 桶内迭代器

- `local_iterator` 和 `const_local_iterator` 继承自全局迭代器。
- 重写了 `operator++`，只在当前桶内移动，不跨越桶边界。
- 用于遍历单个桶内的所有元素。

## 内存管理

### 分配器

- 使用模板参数指定的分配器类型，默认为 `std::allocator<Key>`。
- 内部使用 rebind 机制获取节点分配器和桶分配器。

### RAII

- 在构造函数中分配桶数组，在析构函数中释放所有资源。
- 使用 `destroy_all_nodes()` 释放所有节点。

## 主要接口

### 构造函数

- 默认构造函数：创建空 multiset，默认桶数为 16。
- 指定桶数构造函数：创建空 multiset，使用指定的初始桶数。
- 范围构造函数：从迭代器范围构造。
- 初始化列表构造函数：从初始化列表构造。
- 拷贝/移动构造函数：拷贝或移动另一个 multiset。

### 容量操作

- `empty()`：检查是否为空。
- `size()`：返回元素数量（可大于 1 的键计数）。
- `max_size()`：返回最大可能元素数量。
- `bucket_count()`：返回桶数量。
- `bucket_size(n)`：返回桶 n 中的元素数量。

### 修改器

- `insert(value)`：插入元素，总是成功，返回指向插入元素的迭代器。
- `emplace(args...)`：原位构造元素，总是成功，返回指向插入元素的迭代器。
- `erase(key)`：删除所有等于 key 的元素，返回删除的元素数量（可大于 1）。
- `erase(iterator)`：删除指定位置的元素。
- `merge(other)`：合并另一个 multiset 的所有元素。
- `swap(other)`：交换内容。
- `clear()`：清空所有元素。

### 查找

- `find(key)`：查找等于 key 的元素，返回第一个匹配的迭代器。
- `count(key)`：返回等于 key 的元素数量（可大于 1）。
- `equal_range(key)`：返回等于 key 的元素范围。

### 桶接口

- `bucket(key)`：返回 key 所在的桶索引。
- `begin(n)` / `end(n)`：返回桶 n 的开始/结束迭代器。

### 哈希策略

- `load_factor()`：返回当前负载因子。
- `max_load_factor()`：返回/设置最大负载因子。
- `rehash(n)`：重新哈希，改变桶数量。
- `reserve(n)`：预留空间。

## 与 unordered_set 的关键区别

| 特性 | unordered_set | unordered_multiset |
|------|--------------|-------------------|
| 键唯一性 | 不允许重复 | 允许重复 |
| `insert` 返回值 | `pair<iterator, bool>` | `iterator` |
| `emplace` 返回值 | `pair<iterator, bool>` | `iterator` |
| `count(key)` 返回值 | 0 或 1 | 0 到 N |
| `erase(key)` 返回值 | 0 或 1 | 0 到 N |
| `equal_range(key)` | 最多 1 个元素的范围 | 可能多个元素的范围 |
| `merge` 行为 | 跳过重复键 | 合并所有元素 |
| `operator==` 语义 | 集合相等（元素存在性） | 多重集相等（每个键的计数相同） |
| 插入逻辑 | 检查唯一性，已存在则跳过 | 始终插入，不检查唯一性 |

## 性能考虑

### 时间复杂度

- 插入：平均 O(1)，最坏 O(n)。
- 删除：平均 O(1)，最坏 O(n)。
- 查找：平均 O(1)，最坏 O(n)。
- 重哈希：O(n)。

### 空间复杂度

- 每个元素需要一个节点，包含键、哈希值和指针。
- 桶数组需要 O(bucket_count) 的空间。

### 优化建议

- 使用 `reserve()` 预分配空间，减少重哈希次数。
- 选择合适的负载因子平衡内存和性能。
- 对于大量重复键的场景，unordered_multiset 比 unordered_set 更合适。
